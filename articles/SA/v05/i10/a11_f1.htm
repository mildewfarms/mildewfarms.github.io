Figure 1: Readme file

The idea: present graphs showing multiple server's load from sar output.

How: sa1 and netstat run every 20 minutes on each server of interest
from the sys account crontab. On one machine, the scripts
{cpu,dsk,net}collect.pl run (it is hoped) after the sa1, using NFS
mounts of directory /net/_server_/var/adm/sa to generate a sar text
report for each machine, under control of a perl5 script that reformats
the data for input into gnuplot. The version of gnuplot available to me
does not produce gifs, so I have it create a .pbm file,
which pbmtogif translates.

The data is presented by the web page, servperf.html. In my experience,
Netscape would cache the graphs, and refuse to uncache them (though the
cache clearing options worked fine for non-images). A problem, because
if you look at the graphs at 8am, you want to see newer ones at 5pm.
Netscape would not, however, cache the results of CGI scripts, so I use
the CGI script chooseperf.pl to deliver the gif.
