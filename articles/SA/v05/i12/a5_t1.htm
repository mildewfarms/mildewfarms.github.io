Table 1: Common log files

Name Contents
/var/adm/lastlog userid tty/login-machine came from date time-in time-out use the "last" command to read the file.
/var/adm/message setup in /etc/syslog.conf-might contain critical mail messages, user messages, errors, kernal debugging information, and daemon notices. I prefer "less" or "more" to view.
/var/adm/sulog userids of those who ran the 'su' command, along with the time and date and their tty/login. Use "less" or "more" to view.
/var/adm/wtmp userid on, tty/login, date, from where (host name) - use "who" to view the information contained within.
/var/log/syslog setup in /etc/syslog.conf-might contain mail messages, login logs, and error messages.
