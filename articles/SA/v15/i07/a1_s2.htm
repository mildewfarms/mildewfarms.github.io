HADR and Mon Timing Settings

Most important to the configuration of mon, in a HADR environment is, as we noted previously, the timing values. These must be set correctly so we don't end up in a split-brain scenario. To avoid this, it's important that the interval value in mon.cf for the hadr status and DB2 instance monitor be larger than what you use for "deadtime" in the ha.cf file. This ensures that monitor is not fooled by and will not interfere with a failover.
