Turns and About-Turns

Although the use of static fields for IAC is a logical outcome of the browser implementation as described, Netscape has not been comfortable with that outcome. In Navigator releases 3.0b5-7, changes were introduced to thwart IAC via static fields, with the reason given "to fix real bugs in LiveConnect security where applets could use JavaScript to see data (such as form fields) on HTML pages other than the ones they were associated with" (J. Giannandrea, "Netscape 3.0b7," Posting on java.comp.lang, August 15, 1996). The resulting "outcry" (Netscape's term) obliged it to backtrack and restore this facility. Netscape has decided to introduce a new model of IAC using JavaBeans (J. Giannandrea, Personal e-mail, September 12, 1997).

-- S.B. and J.M.C.
